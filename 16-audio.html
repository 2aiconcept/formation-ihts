Exemples : Balise <audio> HTML5

1. Audio simple avec contrôles

Attribut utilisé : controls

Affiche les contrôles de lecture : play/pause, volume, barre de progression.

2. Musique d'ambiance (autoplay + loop)

Attributs utilisés : autoplay loop muted

autoplay fonctionne uniquement avec muted (son coupé).

L'utilisateur peut ensuite activer le son avec les contrôles.

3. Audio sans contrôles visibles

Un son de notification (invisible, contrôlé par JavaScript)

Sans l'attribut controls, l'audio est invisible.

Utile pour des sons d'effets, notifications, etc. (nécessite JavaScript pour lancer).

4. Compatibilité multi-formats

3 formats fournis : MP3, OGG, WAV

Le navigateur choisit automatiquement le premier format qu'il supporte.

Attributs de la balise <audio>

Attribut Description
controls Affiche les contrôles de lecture (play, pause, volume)
autoplay Lance automatiquement la lecture (nécessite muted)
loop Rejoue l'audio en boucle
muted Audio sans son par défaut
preload Précharge l'audio : none, metadata, auto

Formats audio compatibles

Format Extension Type MIME Caractéristiques
MP3 .mp3 audio/mpeg Universel, le plus compatible
OGG .ogg audio/ogg Open source, bonne qualité
WAV .wav audio/wav Sans perte, fichiers lourds
AAC .aac audio/aac Utilisé par Apple, bonne compression

Bonnes pratiques

  • Toujours fournir au moins 2 formats (MP3 + OGG minimum)
  • Utiliser controls pour que l'utilisateur puisse contrôler la lecture
  • Éviter autoplay sauf pour musique d'ambiance (avec muted)
  • Inclure un message de fallback pour les vieux navigateurs
  • Les fichiers audio peuvent être lourds : optimisez la taille !
