Note importante

autoplay ne fonctionne que si la vidéo est muted (sans son). C'est une règle des navigateurs modernes pour éviter les vidéos bruyantes automatiques.

playsinline est nécessaire sur iOS pour éviter le plein écran automatique.

Vidéo responsive (s'adapte à toutes les tailles d'écran)

Astuce Responsive

En utilisant width: 100% et height: auto, la vidéo s'adapte automatiquement à la taille de l'écran (mobile, tablette, desktop).

Attributs de la balise <video>

Attribut Description Valeurs
controls Affiche les contrôles de lecture (play, pause, volume, etc.) Pas de valeur (présence = activé)
autoplay Démarre automatiquement la lecture Pas de valeur (nécessite muted)
loop Rejoue la vidéo en boucle Pas de valeur
muted Vidéo sans son par défaut Pas de valeur
poster Image affichée avant la lecture URL de l'image
preload Précharge la vidéo none, metadata, auto
width Largeur de la vidéo en pixels Nombre (ex: 800)
height Hauteur de la vidéo en pixels Nombre (ex: 450)
playsinline Lecture sans plein écran sur mobile (iOS) Pas de valeur

Bonnes pratiques

  • Toujours fournir plusieurs formats (MP4 + WebM minimum) pour compatibilité
  • Utiliser poster pour une belle image d'aperçu
  • Ajouter des sous-titres (balise <track>) pour l'accessibilité
  • Éviter autoplay sauf pour vidéos d'arrière-plan sans son
  • Pour responsive : width: 100%; height: auto;
  • Toujours inclure un message de fallback pour vieux navigateurs
